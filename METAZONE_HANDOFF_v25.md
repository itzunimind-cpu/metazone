```sql
-- 1. Create one team_identities row per distinct (user_id, lower(name)) that
--    appears anywhere in `teams` and doesn't already have one.
INSERT INTO team_identities (user_id, canonical_name)
SELECT DISTINCT ON (t.user_id, lower(t.name)) t.user_id, t.name
FROM teams t
WHERE t.user_id IS NOT NULL
  AND NOT EXISTS (
    SELECT 1 FROM team_identities ti
    WHERE ti.user_id = t.user_id AND lower(ti.canonical_name) = lower(t.name)
  )
ORDER BY t.user_id, lower(t.name), t.created_at ASC; -- earliest-seen spelling wins as canonical_name

-- 2. Backfill teams.team_identity_id by case-insensitive name match within owner.
UPDATE teams t
SET team_identity_id = ti.id
FROM team_identities ti
WHERE t.user_id = ti.user_id
  AND lower(t.name) = lower(ti.canonical_name)
  AND t.team_identity_id IS NULL;

-- 3. Sanity check тАФ should return 0 rows. Run this before treating the backfill as done.
SELECT id, name, user_id FROM teams WHERE user_id IS NOT NULL AND team_identity_id IS NULL;
```

**Bug found and fixed during this session's own verification:** the TEAMS/PLAYERS tabs' expand-row toggle functions (`openMergeRow`/`openSplitRow`/`openRosterEditor`) originally used `document.querySelectorAll('[id^="ti-expand-"]')`/`'[id^="pl-expand-"]'` to close any other open row before opening a new one тАФ but this attribute-prefix selector also matched the *inner content* `<div>` (e.g. `ti-expand-body-ti1` also starts with `"ti-expand-"`), not just the intended wrapper `<tr>`, so opening a row set `display:none` on its own content div a moment after populating it, leaving it invisible despite the wrapper `<tr>` correctly showing. ## SECTION 7B тАФ HISTORY.HTML тЖТ DROPPED, SUPERSEDED BY MANAGE.HTML (Session 48)

`history.html` was locked-spec'd early on (cross-tournament match history table, filterable by team/map/stage/event, sortable columns, CSV export) but explicitly deferred "until 2-3 real tournaments of data exist." It was never built, and every page's nav bar linked to it regardless тАФ meaning "ALL TOURNAMENTS" was a dead link on all 7 pages by the time Session 48 started.

**Decision: drop it, repurpose its nav slot for `manage.html` instead.** Reasoning:
- Its core value (cross-tournament match browsing) is already substantially covered elsewhere: `player.html`'s CAREER mode (Session 43) and `analytics.html`'s Routes tab (Session 43) both already browse matches across tournaments.
- A real, currently-unfixable gap existed instead: no way to fix a typo'd team name, merge duplicate opponent `team_identities`, edit a player's IGN, or clean up a roster without deleting an entire tournament. `manage.html` fixes this directly (and implements the previously-flagged FEAT-6 team-identity merge/split tool in the process).
- `manage.html`'s MATCHES tab keeps the one piece of `history.html`'s original spec worth keeping (a cross-tournament match list) тАФ no CSV export or server-side sort/filter in v1, explicitly scoped out rather than silently dropped; easy to add later against the same data if it turns out to matter.

No further action needed on `history.html` тАФ it will not be built.

## SECTION 10 тАФ VOD PAGE ARCHITECTURE (v3.0)

### Boot sequence
1. `getSession()` fast path OR `onAuthStateChange` тАФ both call `loadAllData()` then `applyUrlParams()`
2. `applyUrlParams()` тАФ if URL params present, deep-links to that match. If no params, auto-selects most recently created match by `created_at` desc.
3. `onMatchChange()` тАФ loads shapes, matchPlayers, restores vod_start_offset, auto-fills vod_url, calls `_createYTPlayer()` if URL present, then calls `_postMatchLoadDialogs()`
4. `_postMatchLoadDialogs()` тАФ decides: show editor data dialog (has data, no offset) OR show skip banner (has offset, player ready) OR set `_pendingSkipOffset` (has offset, player still loading)

### YT video sync
- No YT IFrame API dependency тАФ uses postMessage proxy
- `_createYTPlayer(videoId)` creates iframe + proxy object
- `onReady` postMessage sets `_ytPlayerReady = true`, fires `_pendingSkipOffset` if set
- `_ytTimeSyncInterval` polls every 150ms via postMessage to keep `_ytCurrentTime` fresh
- `vodStartOffset` = seconds into video where match starts
- All event timestamps = `matchTime` = `videoCurrentTime - vodStartOffset`

### Key UI elements added in v3.0
- `#skip-to-start-banner` тАФ cyan bar below toolbar, `.show` class toggles visibility

- `#match-status-chip` (editor) тАФ status chip in sidebar, below match selector. Unaffected тАФ editor.html wasn't touched this session.
